# HealthTick 调试指南

## 项目结构说明

HealthTick 是一个纯 Swift Package Manager 项目，没有 `.xcodeproj` 文件。入口在 `Sources/` 目录，通过 `build.sh` 打包成可运行的 macOS app。

---

## 三种调试方式

### 方式一：Xcode（支持断点 / View Debugger）

适合场景：调试 UI 布局、SwiftUI 视图层级、需要打断点单步执行。

```bash
xed .
```

Xcode 自动识别 SPM 包，选择 `HealthTick` scheme 和 My Mac 目标，`Cmd+R` 运行。

**注意：** Xcode 运行的 bundle ID 是 `com.lifedever.healthtick`，与正式版相同，会共用 `~/.health-tick/data.db`。如需隔离数据，运行前先关掉正式版，或提前备份数据库。

---

### 方式二：命令行 debug build（最快迭代）

适合场景：调试业务逻辑、数据库操作、计时器行为，`print()` 输出直接可见。

```bash
swift build && .build/debug/HealthTick
```

- 修改代码后重新执行同一命令即可
- 崩溃时会在终端打印完整 stack trace
- `Ctrl+C` 退出 app

---

### 方式三：Dev 包（与正式版共存，数据隔离）

适合场景：测试完整 app 流程（菜单栏常驻、权限、通知、自动更新等），需要接近真实运行环境。

#### 构建并安装

```bash
./build.sh
```

输出 `Done! Dev app installed to ~/Applications/HealthTick Dev.app` 表示成功。

#### 运行

**推荐方式：直接在终端启动**（print 输出实时可见）

```bash
~/Applications/HealthTick\ Dev.app/Contents/MacOS/HealthTick
```

**或者用 open 启动**（后台运行，需配合 Console.app 看日志）

```bash
open ~/Applications/HealthTick\ Dev.app
```

#### 查看日志（Console.app）

适用于 app 已在后台运行的情况：

1. 打开 Console.app：

```bash
open /System/Applications/Utilities/Console.app
```

2. 菜单栏选择 **Action → Include Info Messages**（否则 debug 级别日志不显示）

3. 搜索栏输入：

```
process:HealthTick
```

#### 数据隔离

Dev 包 bundle ID 为 `com.lifedever.healthtick.dev`，数据库独立存放：

```
~/.health-tick-dev/data.db
```

与正式版的 `~/.health-tick/data.db` 完全隔离，两个版本可同时运行。

清空 dev 数据重新测试：

```bash
rm ~/.health-tick-dev/data.db
```

## 添加日志

### print（简单场景）

```swift
print("[DEBUG] phase: \(phase), remaining: \(remainingSeconds)")
```

### os.log（推荐，Console.app 可分级过滤）

```swift
import os.log

private let logger = Logger(
    subsystem: "com.lifedever.healthtick.dev",
    category: "AppState"
)

logger.debug("work minutes: \(workMinutes)")
logger.info("session started: \(sessionId)")
logger.error("database open failed")
```

在 Console.app 中可以按 `subsystem` 和 `category` 过滤，噪音更少。

## 直接查询数据库

```bash
sqlite3 ~/.health-tick-dev/data.db
```

常用查询：

```sql
-- 查看最近 session 记录
SELECT id, date, work_start, work_end, work_minutes,
       break_actual_seconds, skipped
FROM sessions ORDER BY id DESC LIMIT 20;

-- 查今日工作时长（秒转分钟）
SELECT date,
       SUM(MIN(
           MAX(0, CAST((julianday(work_end) - julianday(work_start)) * 1440 AS INTEGER)),
           work_minutes
       )) AS total_minutes
FROM sessions
WHERE date = date('now', 'localtime') AND work_end IS NOT NULL
GROUP BY date;

-- 查今日打卡记录
SELECT * FROM records WHERE date = date('now', 'localtime');

-- 查配置项
SELECT * FROM config;
```

退出：

```bash
.quit
```

```bash
# 1. 修改代码
# 2. 重新构建并启动
./build.sh && ~/Applications/HealthTick\ Dev.app/Contents/MacOS/HealthTick
```

## 方式对比

| 场景 | 推荐方式 |
|------|----------|
| 打断点 / 单步执行 | 方式一（Xcode） |
| 调业务逻辑 / 数据库 | 方式二（命令行 debug build） |
| 测完整 app 行为 / 菜单栏 | 方式三（dev 包） |
